POST /api/auth/register

Register a new user account with email or phone verification.

Important

After successful registration, users must verify their account using either email or phone verification before they can access protected resources.

Request Parameters
Parameter Type Description
email required string Valid email address
password required string Password (min 8 chars, requires uppercase, lowercase, number)
fullName required string User's full name
verificationMethod required string Verification method ("email" or "phone")
Example Request
curl -X POST https://api.myprofile.ltd/api/auth/register \
    -H "Content-Type: application/json" \
    -d {
        "email": "user@example.com",
        "password": "SecurePass123",
        "fullName": "John Doe",
        "verificationMethod": "email"
    }
Success Response
{
    "success": true,
    "message": "Registration successful. Please verify your email.",
    "userId": "5f9d88b...",
    "verificationMethod": "email",
    "otpRequired": true,
    "otpChannel": "email"
}
Error Response
{
    "success": false,
    "message": "Email address is already registered"
}
POST /api/auth/login

Authenticate a user and receive access tokens.

Security

Access tokens are delivered via HTTP-only secure cookies. The refresh token has a 7-day expiration, while the access token expires in 15 minutes.

Request Parameters
Parameter Type Description
email required string Registered email address
password required string User's password
Example Request
curl -X POST https://api.myprofile.ltd/api/auth/login \
    -H "Content-Type: application/json" \
    -d {
        "email": "user@example.com",
        "password": "SecurePass123"
    }
Success Response
{
    "success": true,
    "user": {
        "id": "5f9d88b...",
        "email": "user@example.com",
        "fullName": "John Doe"
    }
}
Error Response
{
    "success": false,
    "message": "Invalid credentials"
}
